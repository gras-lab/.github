# Security Policy

## Supported Versions

We apply security fixes to the latest version of each active project only.
Older versions are not maintained.

## Reporting a Vulnerability

**Do not open a public GitHub issue for security vulnerabilities.**

Report vulnerabilities privately by emailing one of the maintainers:

- Alfio Spoto — available via GitHub profile
- Gabriele Ruggieri — available via GitHub profile

Include the following in your report:

- A description of the vulnerability and its potential impact
- Steps to reproduce or a proof-of-concept (if available)
- The affected repository and version

## Response Timeline

| Step | Target time |
|---|---|
| Acknowledgement | Within 48 hours |
| Initial assessment | Within 5 business days |
| Fix or mitigation | Depends on severity |
| Public disclosure | After fix is released |

We follow [responsible disclosure](https://cheatsheetseries.owasp.org/cheatsheets/Vulnerability_Disclosure_Cheat_Sheet.html). Credit will be given to reporters unless anonymity is requested.
